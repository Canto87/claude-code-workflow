## Data Flow

### When Question Occurs

```
1. Claude CLI running
2. AskUserQuestion called â†’ CLI output
3. Bridge Server parses output
4. Question pattern detected (options, multiSelect, etc.)
5. Send to web UI via WebSocket
6. Render selection screen on web
7. User selection
8. Send to Bridge Server via WebSocket
9. Bridge Server â†’ CLI stdin
10. CLI continues
```

### When Status Changes

```
1. CLI updates status file (.plan-feature/*/state.md)
2. chokidar detects file change
3. Parse new status
4. Send to web UI via WebSocket
5. Kanban board auto-updates
```

## CLI Output Parsing Example

```javascript
// parser.js
function parseOutput(data) {
  // Detect AskUserQuestion JSON pattern
  const questionMatch = data.match(/\{[\s\S]*"questions"[\s\S]*\}/);
  if (questionMatch) {
    try {
      const parsed = JSON.parse(questionMatch[0]);
      return {
        type: 'question',
        data: parsed.questions[0]
      };
    } catch (e) {}
  }

  // Normal output
  return { type: 'log', content: data };
}
```
